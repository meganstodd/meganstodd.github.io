One Page Design

CONCEPT

A client that owns a restaurant has come to you looking for your website company to redesign the ONE PAGE layout of their current menu page.
Their customers are disinterested by the design of their menu and because of this, not many people want to come to their restaurant. They have provided you with the basic outline which includes the HTML of the site. They are looking for you to redesign it to be more appealing to customers.

THE ACTIVITY

  • In each corner of the room you will find a piece of paper that has an element of a website.
  • In groups, go to the section and list things that fall under that category
  • You only have a few minutes to brainstorm as a group
  • When time is up, continue to the next section and repeat.
  • We will quickly share some of the ideas with the class.

THE TASK

  • Go to this LINK and download the raw upload.html file. Add the file to your presentations/week9 folder in Atom.
  • Using the example menu page as reference, create a mockup of each section of the menu and export them as separate PNG files.
  • Using the upload.html file provided, insert your PNG images into the existing HTML.
  • Link the navigation bar items to each of your PNGs using page anchors. By doing this you are ensuring that your site is a ONE PAGE DESIGN.

REQUIREMENTS

  1. Create a mockup for each section of the restaurant menu. Using at least three pictures, and the font of your choosing.
  2. Use page anchors to link each nav item to the different menu sections of the page.
  3. Have a cohesive theme.
  4. Add links to the Nav tag, and style how you would like.
  5. Add a header and style as you would like.
  6. Be creative & have fun!

TIP!

  • You can create multiple art boards in Illustrator and Photoshop, each with a different section of the menu, and then export them all separately as PNGs

UPLOAD

  • When complete, link the file to yourmain index.html page.
  • Then commit and push to origin.

RUBRIC

  • Create a PNG mockup for every part of the menu - 0/3
  • Overall appearance and Aesthetic - 0/5
  • Page Anchors and Functionality - 0/2
  • TOTAL: 0/10
